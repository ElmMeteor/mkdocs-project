    脚注（Footnote）¶

    脚注简介

    脚注 footnote 允许在正文中插入批注、引用、解释等补充说明。 支持数字编号或自定义命名，且可多次复用。 适合学术、教程、批量引用场景。 ⚠️ 脚注语法并非原生 Markdown 标准，需要启用扩展（如 Python-Markdown、MkDocs Material 等）才可使用。

    数字脚注（自动或手写编号）

    用法示例

    牛顿提出万有引力定律[^1]，这一理论影响深远[^1]。

[^1]: 经典物理学的重要里程碑。

    牛顿提出万有引力定律1，这一理论影响深远1。

    • 自动编号，适合简单引用
    • 可多次引用，底部只出现一条

    命名脚注（自定义名称）

    用法示例

    蟒蛇 Python 是流行的编程语言[^py]，常用于数据分析[^py]。

[^py]: Python，由荷兰程序员 Guido van Rossum 开发。

    蟒蛇 Python 是流行的编程语言2，常用于数据分析2。

    • 支持英文/下划线/单词等，便于分类和语义化管理
    • 适合批注场景，如同义词、名词解释等多次统一说明

    混合批量脚注

    用法示例

    名词解释请见[^dict]；原理推导见论文[^2]。

[^dict]: 术语表页面。
[^2]: Smith, A. (2020), Theoretical Analysis.

    名词解释请见3；原理推导见论文4。

    • 数字和命名脚注可混用，便于后期维护整理

    脚注批量复用示例

    用法示例

    水是生命之源[^water]，空气同样重要[^air]。环境保护应注重水[^water]与空气[^air]的治理。

[^water]: 见联合国水资源白皮书（2023）
[^air]:   见 WHO 环境空气质量标准

    水是生命之源5，空气同样重要6。环境保护应注重水5与空气6的治理。

    • 引用无上限，只需写一条内容即可

    易错区分与最佳实践

    脚注类型与管理

    用法 适合场景 是否支持多处复用 管理便利 例子
    [^1] 文献脚注 是 一般 牛顿理论1
    [^word] 术语批注 是 高 Python 脚注2
    [^long_note] 长说明/FAQ 是 很高 解释同义词[^same_term]
    [数字+字母] 特殊编号 是 高 第一章脚注[^ch1_ref]，第二章脚注[^ch2_ref]
    • 推荐批量复用场景用易记英文名如 [^python]、[^faq1]、[^说明1]
    • 不推荐用非 UTF-8 字符做标识符（如 emoji 等可能不被部分渲染器支持）
    • 不同脚注内容不可同名，同名会自动合并

    • 内容可包含链接、代码、图片、列表等 Markdown 元素

    FAQ

    多个脚注是否能合并？

    编号/名称相同自动合并到一条，内容显示在脚注区。

    脚注格式错误会怎样？
    • 错误例：[^1:] ← 少了空格或冒号方向
    • 脚注标记名不能与已有链接等重复，否则会覆盖或冲突
    命名脚注和普通有啥区别？
    • 只是命名方式不同，都可以批量复用，同样可以多次引用
    • 推荐命名（而非纯数字）以增强维护性和可读性

    最佳实践

    写学术、科普、教程时，统一用命名脚注和批量复用，能让读者更容易找到本页所有“补充说明”。
