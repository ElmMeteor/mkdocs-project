# Markdown 脚注命名与复用说明

**Markdown 脚注 (footnote) 支持数字型或自定义命名，可任意多次复用。了解脚注不同写法，有助于批量标注引用、解释术语、增加阅读专业性。**

---

## 1. 数字脚注（自动或手写编号）

```markdown
牛顿提出万有引力定律[^1]，这一理论影响深远[^1]。

[^1]: 经典物理学的重要里程碑。
```
效果：  
牛顿提出万有引力定律[^1]，这一理论影响深远[^1]。

[^1]: 经典物理学的重要里程碑。

- **优点**：自动编号，适合数量较少、简单引用。
- **可多次引用**：多处用同标号，底部只出现一条。

---

## 2. 命名脚注（自定义名称）

```markdown
蟒蛇 Python 是流行的编程语言[^py]，常用于数据分析[^py]。

[^py]: Python，由荷兰程序员 Guido van Rossum 开发。
```
效果：  
蟒蛇 Python 是流行的编程语言[^py]，常用于数据分析[^py]。

[^py]: Python，由荷兰程序员 Guido van Rossum 开发。

- **优点**：支持英文/下划线/单词等，便于分类和语义化管理。
- **适合批注场景**：如同义词、名词解释等处多次出现统一说明。

---

## 3. 混合批量脚注

你可以数字和命名脚注混合：

```markdown
名词解释请见[^dict]；原理推导见论文[^2]。

[^dict]: 术语表页面。
[^2]: Smith, A. (2020), Theoretical Analysis.
```
- 用于不同类型脚注混用，便于后期维护整理。

---

## 4. 脚注批量复用示例（都只写一条内容即可，引用无上限）

```markdown
水是生命之源[^water]，空气同样重要[^air]。环境保护应注重水[^water]与空气[^air]的治理。

[^water]: 见联合国水资源白皮书（2023）
[^air]:   见 WHO 环境空气质量标准
```

---

## 5. 易错区分与最佳实践

| 用法             | 适合场景   | 是否支持多处复用 | 管理便利 | 例子                                        |
|------------------|------------|------------------|----------|---------------------------------------------|
| `[^1]`           | 文献脚注   | 是               | 一般     | 牛顿理论[^1]                                |
| `[^word]`        | 术语批注   | 是               | 高       | Python 脚注[^py]                            |
| `[^long_note]`   | 长说明/FAQ | 是               | 很高     | 解释同义词[^same_term]                      |
| `[数字+字母]`    | 特殊编号   | 是               | 高       | 第一章脚注[^ch1_ref]，第二章脚注[^ch2_ref]  |

- 推荐批量复用场景用易记英文名如 `[^python]`、`[^faq1]`、`[^说明1]`。  
- 不推荐用非 UTF-8 字符做标识符（如 emoji 等可能不被部分渲染器支持）。
- 不同脚注内容不可同名，同名会自动合并。

---

## 6. 脚注内容的高级用法

你脚注内容可以包含任何 Markdown，支持**链接、代码、图片，甚至列表**：

```markdown
脚注高级用法[^demo]。

[^demo]: 
    - 支持多行内容
    - 可写
      ```
      代码块
      ```
    - 也可加 [链接](https://www.example.com)
```

---

## 7. FAQ

??? tip "多个脚注是否能合并？"
    编号/名称相同自动合并到一条，内容显示在脚注区。

??? warning "脚注格式错误会怎样？"
    - 错误例：`[^1:]` ← 少了空格或冒号方向
    - 脚注标记名不能与已有链接等重复，否则会覆盖或冲突

??? info "命名脚注和普通有啥区别？"
    - 只是命名方式不同，**都可以批量复用，同样可以多次引用**
    - 推荐命名（而非纯数字）以增强维护性和可读性

---

!!! tip "最佳实践"
    写学术、科普、教程时，统一用命名脚注和批量复用，能让读者更容易找到本页所有“补充说明”。
